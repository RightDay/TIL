# 13. 컨텍스트 스위칭

CPU에 실행할 프로세스를 교체하는 기술이다.



## 컨텍스트 스위칭의 순서

1. 실행 중지할 프로세스 정보를 해당 프로세스의 PCB에 업데이트해서, 메인 메모리에 저장한다.
2. 다음 실행할 프로세스 정보를 메인 메모리에 있는 해당 PCB 정보를 CPU에 넣고 실행한다.



### PCB(Process Control Block, Prcess Context Block)

여러 process의 주요 상태(process ID, Register 값(PC, SP), Scheduling Inflo, Memory Info 등)을 PCPB는 프로세스가 실행 중인 상태를 캡처/구조화해서 저장한다.

PCB가 process의 상태 정보를 관리하고, 이를 컨텍스트 스위칭, 스케줄링 등에 사용한다.

> 디스패치 (dispatch) : ready 상태의 프로세스를 running 상태로 바꾸는 것



## 참고

컨텍스트 스위칭은 굉장히 짧은 시간(수십ms)단위로 발생하며 이 때 오버헤드가 발생한다. 그래서 이 기술은 어셈블리어로 구현하여 실행속도를 극대화한다.

초기 컴퓨터 프로그램들은 어셈블리어로 작성되어 있어, 서로 다른 CPU 아키텍처가 등장할 때마다 매번 작성해야 하고 프로그램 작성 속도가 매우 떨어진다는 단점이 있다.

하지만 컴파일러의 등장으로 CPU 아키텍처에 따라서 컴파일러 프로그램만 만들면 되기 때문에 기존 코드를 재작성할 필요가 없도록 변경되었다. 대신 어셈블리어로 작성한 코드보다 속도가 떨어질 수 있다.

컴파일러 등장

- CPU 아키텍처에 따라서는 컴파일러 프로그램만 만들면 된다. 기존 크드는 재작성할 필요 없다.
- 그러나, 어셈블리어로는 작성한 코드보다는 속도가 떨어질 수 있다.

리눅스의 경우 컨텍스트 스위칭 코드는 각 CPU마다 별도로 존재한다.



## 정리

컨텍스트 스위칭

- 프로세스 상태 정보를 PCB로부터 CPU에 로드하고 실행한다.
