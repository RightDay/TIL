# DirectX11 Basic

## Direct3D의 개요

- 응용 프로그램이 3차원 그래픽 가속 기능을 이용해서 **3차원 세계를 렌더링할 수 있게 하는 저수준 API**이다.
- **그래픽 하드웨어를 제어**할 수 있는 **소프트웨어 인터페이스를 제공**한다.
- Direct3D 11 대응 그래픽 장치는 반드시 **Direct3D 11 능력 집합(capability set) 전체를 지원해야 한다.**

## COM(Component Object Model)

- DirectX의 프로그래밍 **언어 독립성과 하위 호환성을 가능하게 하는 기술**이다.
- COM의 **세부사항 대부분은 프로그래머에게 드러나지 않는다.**
- **COM 인터페이스로의 포인터를 특별한 함수들** 또는 다른 **COM 인터페이스의 메서드를 이용해서 얻는 방법이다.**
- COM 인터페이스는 **new 키워드로 생성하지 않는다.**
- 인터페이스를 다 **사용하고 난 후에는 인터페이스의 Release 메서드를 호출해 주어야 한다.**
  - 모든 COM 인터페이스는 **IUnknown이라는 COM 인터페이스의 기능을 상속**받으며, 그 **인터페이스가 Release라는 메서드를 제공**한다.
  - 이는 **COM 객체들이 자신만의 고유한 방식으로 메모리를 관리**하기 때문이다.

## 텍스처 및 자료 자원 형식

- 텍스처는 흔히 이미지 자료의 저장을 떠올리지만, 실제로는 그보다 범용적이다.
- 텍스처는 단순한 자료 배열이 아닌 밉맵 수준들이 존재할 수 있으며, GPU는 필터링 다중표본화 등의 특별한 연산을 텍스처에 적용할 수 있다.
- 텍스처에는 특정 형식(format)을 따르는 자료만 담을 수 있다.
  - 구체적인 형식은 DXGI_FORMAT이라는 열거형으로 지정한다.

## 교환 사슬과 페이지 전환

- 전면 버퍼(front buffer) : **화면에 표시되는 이미지 자료를 담는다.**

- 후면 버퍼(back buffer) : **전면 버퍼가 화면에 표시되는 동안 다음 표시될 이미지 자료를 그리는 곳**이다.

- ### 후면버퍼를 전면 버퍼와 교환하는 과정

  1. 전면버퍼가 **화면에 표시되어 있는 동안** 애니메이션의 **다음 프레임을 후면 버퍼에 그린다.**
  2. 전면 버퍼와 후면 버퍼의 **역할을 바꾼다.**
  3. 이를 제시(presenting)이라 부른다.

- 전면 버퍼와 후면 버퍼는 하나의 **교환사슬(swap chain)을 형성한다.**

- 두 개의 버퍼(전면과 후면)를 사용하는 것을 **이중 버퍼링(double buffering)이라고 부른다.**

## 깊이 버퍼링

- 깊이 버퍼(depth buffer)는 **이미지 자료를 담지 않는 텍스처**이다.
- 깊이 버퍼는 **각 픽셀의 깊이 정보를 담는다.**
- 픽셀의 깊이는 0.0에서 1.0까지의 값으로 **0.0은 관찰자의 최대한 가까운 픽셀**에 해당하고 **1.0은 최대한 먼 픽셀**에 해당한다.
- **깊이 버퍼의 원소들**과 **후면 버퍼의 픽셀들**은 **일대일로 대응**된다.
- Direct3D는 **깊이 버퍼링** 또는 **z-버퍼링**이라는 기법을 사용한다.
- 깊이 버퍼링에서 **물체들을 그리는 순서는 중요하지 않다.**
- **렌더링되는 각 픽셀의 깊이 값을 계산해서 깊이 판정을 수행**한다. 깊이 판정은 **후면 버퍼의 특정 픽셀 위치에 기록될 픽셀들의 깊이들을 비교**한다.
- 깊이 값을 비교했을 때 **관찰자에게 가장 가까운 픽셀이 승자가 되어서 후면 버퍼에 기록**된다.
- 깊이 버퍼도 하나의 텍스처이므로 **특정한 자료 형식을 지정해서 생성**해야 한다.

## 텍스처 자원 뷰

- **텍스처를 렌더 대상으로 묶는 것**(Direct3D가 텍스처에 렌더링하는 경우)과 **셰이더 자원으로서 묶는 것**(셰이더 안에서 텍스처를 추출하는 경우) 이 두 가지 용도의 텍스처를 생성할 때 다음과 같이 **텍스처를 묶을 두 파이프라인 단계를 지정한 결속 플래그를 사용한다.**

  - ```cpp
    D3D11_BIND_RENDER_TARGET | D3D11_BIND_SHADER_RESOURCE
    ```

- **텍스처가 연관된 자원 뷰를 파이프라인 단계에서 묶는다.**

- Direct3D에서 텍스처를 사용하려면 **텍스처의 초기화 시점에서 텍스처의 자원 뷰(resource view)를 생성해야 한다.**

  - 이렇게 하면 **실행시점 모듈(런타임)과 구동기(드라이버)가 유효성 점검과 매핑을 뷰 생성 시점에서 수행**할 수 있기 때문에 **결속 시점에서의 형식 점검이 최소화**된다.

- 텍스처를 **렌더 대상과 셰이더 자원으로 사용**하기 위해서 **두 개의 뷰를 생성할 필요**가 있다.

  - 렌더 대상 뷰(ID3D11RenderTargetView), 셰이더 자원 뷰(ID3D11ShaderResourceView)

- 자원 뷰들은 본질적으로 두 가지 일을 한다.

  1. Direct3D에게 **자원의 사용 방식(파이프라인의 어떤 단계에 묶을 것인지)을 알려준다.**
  2. 생성 시점에서 **무형식을 지정한 자원 형식의 구체적인 형식을 결정**한다.

## 다중표본화의 이론

- 선을 픽셀들로 배열로 근사할 때 생기는 \'계단 현상\'을 앨리어싱(aliasing)이라고 한다.
- 앨리어싱 제거(antialiasing) 기법 중 **초과표본화(supersampling\)**
  - 후면 버퍼와 깊이 버퍼를 화면 해상도보다 4배(가로, 세로 두배 씩) 크게 잡는다. 
  - **3차원 장면을 4배 크기의 해상도에서 후면 버퍼에 렌더링**하고, 이미지를 화면에 제시할 때 후면 버퍼를 원래 크기의 버퍼로 환원(resolving)한다.(하향표준화(downsampling)라고도 부른다.)
  - **4픽셀의 네 색상의 평균을 최종 색상으로 사용한다.**
- Direct3D는 다중표본화(multisamppling)라는 절충적인 앨리어싱 제거 기법을 지원한다.
  - 다중 표본화는 일부 계산 결과를 부분픽셀들 사이에서 공유하기 때문에 초과표본화보다 비용이 적다.
  - 4X 다중 표본화(픽셀당 부분픽셀 4개)의 경우 **초과표본화처럼 크기가 화면 해상도 4배인 후면 버퍼와 깊이 버퍼를 사용한다.**
  - 다중표본화는 이미지 색상을 각 부분픽셀마다 계산하는 것이 아니라 **픽셀당 한 번만 계**산하고(픽셀의 중심에서), 그 색상과 **부분픽셀들의 가시성**(부분 픽셀당 깊이·스텐실 판정이 일어남)과 **포괄도**(부분픽셀을 다각형이 어느 정도나 덮고 있는지를 뜻하는 값)를 이용해서 **최종 색상을 결정한다.**

## Direct3D의 다중표본화

- DXGI_SAMPLE_DESC라는 구조체를 채워야 한다.

  - ```cpp
    typedef struct DXGI_SAMPLE_DESC
    {
        UINT Count;
        UINT Quality;
    } DXGI_SAMPLE_DESC, *LPDXGI_SAMPLE_DESC;
    ```

    - Count 멤버는 픽셀당 추출한 표본의 개수를 지정한다.
    - Quality 멤버는 원하는 품질 수준(quality level)을 지정한다.

- **품질 수준들의 범위는 텍스처 형식과 픽셀당 표본 개**수의 의존한다.

  - ```cpp
    HRESULT ID3D11Device::CheckMultisampleQualityLevels(
    	DXGI_FORMAT Format, UINT SampleCount, UINT *pNumQualityLevels);
    ```

    - 하나의 텍스처 형식, 표본 개수 조합에 대한 유효한 품질 수준의 범위는 0에서 *pNumQualityLevels - 1까지이다.

- 한 픽셀 당 추출할 수 있는 최대 표본 개수

  - ```cpp
    #define D3D11_MAX_MULTISAMPLE_SAMPLE_COUNT ( 32 )
    ```

- **다중표본화의 성능 및 메모리 비용을 합리적인 수준으로 유지하기 위해 표본을 4개나 8개만 추출**하는 경우가 많다.

- 다중화표본화를 사용하고 싶지 않으면 표본 개수를 1로, 품질 수준을 0으로 설정하면 된다.

- 모든 Direct3D 11 대응 장치는 **모든 렌더 대상 형식에 대해 4X 다중표본화를 지원**한다.

## 기능 수준

- D3D_FEATURE_LEVEL이라는 열거형으로 대표되며, 대략적으로 버전 9에서 11까지의 여러 Direct3D 버전들에 대응된다.
- 사용자의 **하드웨어가 특정 기능 수준을 지원하지 않는다면**, 그보다 **더 낮은(오래된) 기능 수준으로 내려가서 응용 프로그램을 실행**한다.
