# 루프와 관계 표현식

- ## for 루프

  - ### for 루프의 각 부분

    - for 루프는 단계적인 처리를 통해 반복 작업을 수행한다.

    - #### for 루프의 단계

      1. 조건 검사에 사용할 카운터 값을 초기화한다.
      2. 루프를 진행할 것인지 조건을 검사한다.
      3. 루프 몸체를 수행한다.
      4. 카운터 값을 갱신한다.

    - 초기화(initialization), 조건 검사(test), 갱신(update)을 처리하는 for 루프의 세 제어 부분은 괄호로 묶여 있다.

      - 각각 하나의 표현식이며. 세미콜론(;)에 의해 분리되어 있다.

    - 제어 부분 뒤에 오는 구문을 루프 몸체(body)라 한다.

    - 루프 몸체는 조건 검사가 참일 경우에만 수행된다.

      - ```cpp
        for (initialization; test-expression; update-expression)
            body
        ```

    - 몸체 안에 여러 개의 구문이 사용되었을 경우에도 for 루프 전체를 하나의 구문으로 간주한다.

    - 초기화 표현식은 어떤 변수에 시작값을 설정한다. 

      - 초기화는 처음에 단 한번만 수행한다.
      - 루프의 주기를 카운트하는 데 그 변수를 사용한다.

    - test-expression은 루프 몸체 body를 실행할 것인지 여부를 결정한다.

      - test-expression은 두 개의 값을 서로 비교하는 관계식이다.
      - test-expression이 참이면 body가 실행된다.
      - test-expression에는 참-거짓을 비교하는 관계식뿐만 아니라, 어떠한 표현식도 사용할 수 있다.
      - C++는 그것을 bool형으로 변환한다.
        - 0 값을 가진 표현식이면 bool형 값 false로 변환되고 루프가 종료된다.
        - 표현식이 0이 아닌 다른 수로 평가되면 bool형 값 true로 변환되고 루프가 계속 진행된다.

    - for 루프는 진입 조건(entry-condition) 루프이다.

      - 따라서 각 루프 주기에 진입할 때 조건 검사 표현식이 평가된다.
      - 조건 검사 표현식이 거짓으로 평가되면 루프 몸체는 결코 수행되지 않는다.
      - 진입 조건 루프는 루프를 실행하기 전에 조건을 먼저 검사하기 때문에 프로그램이 문제를 일으키는 것을 막을 수 있다.

    - update-expression은 루프 몸체가 실행되고 나서 루프 주기의 끝에서 평가된다.

      - 일반적으로 update-expression은 루프 주기의 수를 추적하는 변수의 값을 증가시키거나 감소시킨다.
      - 어떠한 표현식도 update-expression으로 사용할 수 있다.
