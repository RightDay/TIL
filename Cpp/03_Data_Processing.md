# 데이터 처리

* C++에 내장된 데이터형에는 기본형과 복합형이 있다.

* 기본형에는 정수를 표현할 수 있는 정수형과 소수부가 있는 수를 표현할 수 있는 소수점형이 있다.




* ## 간단한 변수

  * ### 변수 이름

    * C++에서는 의미를 쉽게 알 수 있는 변수 이름을 사용할 것을 권장한다.
    * 변수 이름을 짓는 규칙
        * 변수 이름에는 영문자, 숫자, 밑줄(_)만 사용할 수 있다.
        * 숫자를 변수 이름의 첫 문자로 사용할 수 없다.
        * 변수 이름에서 대문자와 소문자는 구별된다.
        * C++의 키워드는 변수 이름으로 사용할 수 없다.
        * 두 개의 밑줄 문자로 시작하는 이름이나, 밑줄 문자와 대문자로 시작하는 이름은, 그것을 사용하는 컴파일러와 리소스가 사용하기로 예약되어 있다.
        * 하나의 밑줄 문자로 시작하는 이름은, 그것을 사용하는 컴파일러와 리소스가 전역 식별자(global identifier)로 사용하기로 예약되어 있다.(컴파일 에러 X)
        * 변수 이름의 길이는 제한이 없으며, 변수 이름에 쓰인 모든 문자들이 유효하다. 그러나, 어떤 플랫폼은 고유의 길이 제한이 있다.

  

  * ### 정수형

    * 소수부가 없는 수이다.
    * 컴퓨터의 메모리 용량에 한계가 있어 정수들의 부분 집합만을 나타낼 수 있다.
    * C++는 여러 가지 정수형을 제공한다.

  

  * ### 비트와 바이트

    * #### 비트(bit)

      * 컴퓨터의 메모리를 구성하는 기본 단위는 비트이다.
        * ex) 전기 스위치가 꺼진 상태는 값 0, 켜진 상태는 값 1을 나타낸다.
      * 8비트 폭의 메모리는 256가지(2^8)의 비트 조합을 만들 수 있다.

    * #### 바이트(byte)

      * 8비트의 메모라 단위를 의미한다.
      * 바이트 단위로 컴퓨터의 메모리 크기를 나타낼 때, 1킬로바이트는 1024바이트이고 1메가바이트는 1024킬로바이트이다.
      * C++에서 바이트는 컴파일러의 기본 문자 세트를 수용할 수 있는 최소한의 연속된 비트들로 구성된다.
        * 바이트를 구성하는 비트들의 조합으로 나타낼 수 있는 값의 수가, 컴파일러가 사용할 수 있는 문자 수와 같거나 그보다 커야 한다.
        * ASCII와 EBCDIC 문자 세트를 사용하는 시스템에서 C++의 바이트는 8비트이다.
        * Unicode같이 더 큰 문자 세트를 사용하는 시스템에서는 C++의 바이트는 16비트 또는 32비트이다.

  

  * ### short, int, long, long long 정수형

    * short, int, long은 정수를 저장하는 데 사용하는 비트 수가 다르다.

    * 컴퓨터마다 데이터형이 동일한 폭을 갖고 있지 않아, C++는 데이터형들의 최소 크기만을 정하여 표준을 제공한다.

      * short형은 최소한 16비트 폭을 가진다.
      * int형은 최소한 short만큼 크다.
        * 컴퓨터마다 폭이 16, 24, 32비트가 될 수 있다.
      * long형은 최소한 32비트 폭을 가지며,  최소한 int만큼은 크다.
      * long long형은 최소한 64비트 폭을 가지며 최소한 long만큼은 크다.

    * #### climits에 정의되어 있는 기호 상수들

      * CHAR_BIT, SHRT_MAX, INT_MAX, INT_MIN, LONG_MAX 등

  

  * ### unsigned형

    * 음의 정수값을 저장할 수 없는 변형이다.

    * 변수에 저장할 수 있는 최대값을 늘릴 수 있다.

      * ex) short형 -32768에서 +32767까지의 범위를 unsigned short는 0부터 65535까지의 범위를 갖는다.

    * ```cpp
      unsigned short change;	//unsigned short형
      unsigned int rovert;	//unsigned int형
      ```

  

  * ### 정수형 정하기

    * 일반적으로 int형은 컴퓨터가 가장 효율적으로 처리하는 정수 형식이다.
    * 메모리 절약이 중요한 경우에는 int형 대신 short형을 사용한다.

  

  * ### 정수형 상수

    * 212, 1667과 같이 프로그램에 직접 써 넣는 정수를 말한다.

    * 10진수, 8진수, 16진수로 정수형 상수를 프로그램에 직접 작성할 수 있다.

    * ```cpp
      int chest = 42;		//10진 정수형 상수
      int waist = 0x42;	//16진 정수형 상수
      int inseam = 042;	//8진 정수형 상수
      ```

      * chest 값 : 42
      * waist 값 : 66
      * inseam = 34

    * 이러한 표기법들은 표기를 쉽게 하기 위한 것이다.

    * 어느 것으로 표기하더라도 컴퓨터 내부에서는 모두 동일한 2진수 값으로 처리된다.

    * 어떤 정수값을 16진수나 8진수 형태로 출력하려면 출력하라는 메시지를 cout에 전달하는 dec, hex, oct 조정자를 제공한다.(10진수가 디폴트 형태이다.)

      * ```cpp
        int a = 42;
        
        cout << a << " (10진수)" << endl;
        cout << hex;	//16진법으로 바꾸는 조정자
        cout << a << " (16진수)" << endl;
        cout oct;		//8진법으로 바꾸는 조정자
        cout << a << " (18진수)" << endl;
        ```

        * 42 (10진수)
        * 2a (16진수)
        * 52 (8진수)

  

  * ### C++가 상수의 데이터형을 결정하는 방법

    * C++는 특별한 이유가 없다면 정수형 상수를 모두 int형으로 저장한다.
    * 그러나 특정 데이터형을 의미하는 접미어를 상수에 붙였을 때와, 값이 너무 커서 int형으로 저장할 수 없을 때에는 그렇지 않다.
      * l이나 L은 long형을 의미, u나 U는 unsigned int형을 의미한다.

  

  * ### char형: 문자와 작은 정수

    * 문자와 숫자를 저장한다.
    
    * 문자를 수치 코드로 나타낸다.
    
    * 1바이트이다.
    
    * 가장 많이 사용하는 문자 세트는 ASCII이다.
    
    * #### 확장 char형: wchar_t(wide character type)
    
      * 프로그램이 1바이트(8비트)로 표현할 수 없는 문자 세트(일본어, 중국어, 한국어)를 처리해야 할 경우 사용한다.
    
      * 시스템에서 사용되는 가장 큰 확장 문자 세트를 나타낼 수 있다.
    
      * 확장 문자 상수나 확장 문자 문자열은 그 앞에 L을 붙여서 나타낸다.
    
        * ```cpp
          wchar_t bob = L'P';	//확장 문자 상수
          wcout << L"tall" << endl;	//확장 문자 문자열의 출력
          ```

  

  * ### bool형

    * 참이나 거짓 중 어느 한 가지 값을 가진다.

    * C++는 0이 아닌 값을 참, 0인 값을 거짓으로 해석한다.

    * ```cpp
      bool start = -100;	//start에 true가 대입된다.
      bool stop = 0;		//stop에 false가 대입된다.
      ```

  

  * ### const 제한자

    * const 키워드를 사용하여 변수를 선언하고 초기화하면 변수를 상수로 사용할 수 있다.

      * ```cpp
        const int MONTHS = 12;	//MONTHS는 12를 나타내는 기호 상수이다.
        ```

    * 한번 초기화하면 그 값이 고정되어 컴파일러가 값을 변경하려는 어떠한 시도도 허용하지 않는다.

    * const 상수는 선언할 때 값으로 초기화하지 않으면 변경할 수 없는 미확정 값으로 남겨진다.

      * ```cpp
        const int toes;	//이 시점에서 toes의 값은 미확정
        toes = 10;		//늦게 초기화함.
        ```

    * #### #define문보다 cosnt 사용하는 것이 좋은 이유

      1. 데이터형을 명싲거으로 지정할 수 있다.
      2. C++의 활동 범위 규칙(scoping rules)에 의해 그 정의를 특정 함수나 파일에서만 사용할 수 있도록 제한할 수 있다.
      3. 배열이나 구조체와 같은 보다 복잡한 데이터형에도 const를 사용할 수 있다는 점이다.

  

* ## 부동 소수점수

  * 소수부가 있는 수들을 나타낼 수 있다.

    * 한 부분은 기본값을 나타내고, 다른 한 부분은 기본값을 키우거나 줄이는 스케일을 나타낸다.
    * ex) 34.1245와 34124.5의 기본값과 스케일
      * 34.1245의 기본 값 : 0.341245 / 스케일 : 100
      * 34124.5의 기본 값 : 0.341245 / 스케일 : 100000
    * 소수점이 스케일에 따라 자리를 옮기기 때문에 이 데이터형을 부동 소수점형이라고 부른다.
    * C++에서는 스케일 인수가 10이 아닌 2이다.
  * 매우 큰 수, 매우 작은 수를 나타낼 수 있으며 내부적으로 정수형과는 다른 방식으로 나타낸다.

  

  * ### 부동 소수점수의 표기

    * #### 소수점 표기법

      * ```
        12.34
        939001.32
        00.0023
        8.0
        ```

      * 8.0과 같이 소수부가 비록 0이라 할지라도 소수점이 있기 때문에 8.0은 정수형이 아닌 부동 소수점형으로 표현된다.

    * #### 지수 표기(Enotation)

      * ex) 3.45E6은 값 3.45에 1,000,000을 곱하라는 뜻이다.

        * E6은 10의 6제곱을 의미하므로 3.45E6은 3,450,000이 된다.
        * 여기서 6을 지수(exponent), 3.45를 가수(mantissa)라 한다.

      * ```
        2.52e+8	// E 또는 e를 사용할 수 있으며 +는 생략할 수 있다.
        8.33E-4	// 지수는 음수일 수도 있다.
        7E5		// 7.0E+0.5와 같다.
        -18.32e13	//앞에 +나 -부호를 넣을 수 있다.
        ```

      * 매우 큰 수나 매우 작은 수를 나타내는 데 무척 유용하다.

      * 소수점이 사용되지 않더라도 항상 그 수를 부동 소수점형으로 저장한다.

      * E 또는 e를 사용하 수 있고, 지수는 + 또는 - 부호를 가질 수 있다.

      * 지수는 + 또는 - 부호를 가질 수 있다.

      * 음의 지수를 사용하는 것은 10의 거듭제곱을 곱하는 것이 아니라, 10의 거듭제곱으로 나누는 것이다.

        * ex) 8.33E-4는 8.33/10^4 또는 0.000833을 의미한다.

      * d.dddE+n 형식은 소수점을 오른쪽으로 n자리만큼 이동한다.

      * d.dddE-n 형식은 소수점을 왼쪽으로 n자리만큼 이동한다.
  
  
  
  * ### 부동 소수점형
  
    * float, double, long double의 세 가지 부동 소수점형이 있다.
  
      * 각 데이터형들은 유효 숫자의 개수와 지수의 최소 허용 범위가 다르다
  
        * 유효 숫자 : 어떤 수에서 의미 있는 숫자
  
          * ex) 백두산의 높이가 2745미터라고 말한다면 유효 숫자 개수는 4
  
            2700미터라고 말한다면 유효 숫자의 개수는 2가 된다.
  
    * | 데이터형    | 크기            |
    | ----------- | --------------- |
      | float       | 32비트          |
      | double      | 64비트          |
      | long double | 80, 96, 128비트 |
    
    * 일반적으로 cout은 뒤에 붙은 0을 출력하지 않는다.
    
    * cout.self()를 호출하면 0을 출력한다.
  
  
  
  * ### 부동 소수점형 상수
  
    * 기본적으로 double형으로 저장된다.
  
    * float형으로 저장하고 싶으면 f나 F를 접미어로 사용한다.
  
      * ```cpp
        1.234f	// float형 상수
        ```
  
    * long double형으로 저장하고 싶으면 l이나 L을 접미어로 사용한다.
  
      * ```cpp
        2.2L	// long double형 상수
        ```
  
  
  
  * ### 부동 소수점수의 장점과 단점
  
    * 장점
  
      * 정수와 정수 사이에 있는 값을 나타낼 수 있다.
      * 스케일을 사용하여 매우 큰 범위의 값을 나타낼 수 있다.
  
    * 단점
  
      * 연산 속도가 수치 연산 보조 프로세서(math coprocessor)가 없는 컴퓨터에서 정수 연산보다 속도가 느리다.
  
      * 정밀도를 잃을 수 있다.
  
        * ```cpp
          float a = 2.34E+22f;
          float b = a + 1.0f;
          
          cout << "a = " << a << endl;
          cout << "b - a = " << b - a << endl;
          ```
  
        * b - a의 예상되는 결과는 1이어야 하지만 결과는 b - a = 0이다.
  
        * 2.34E+22가 소수점 위로 23개의 숫자를 가진 큰 수이기 때문이다.
  
          * 이 수에 1을 더하는 것은 23번째 숫자에 1을 더하는 것과 같다.
  
          * float형은 처음 6개의 숫자 또는 7개의 숫자까지만 나타낼 수 있으므로, 23번째 숫자에 1을 더하는 것은 아무런 효과도 갖지 못한다.
  
            
  
  * ### 데이터형의 분류
  
    * singed 정수형 : signed char, short, int, long
  
    * unsigned 정수형 : unsigned char, short, int long
  
    * 부동 소수점형 : float, double, long double
  
    * 정수형과 부동 소수점형을 합쳐 산술형(arithmetic type)이라고 부른다.
  
      
  
* ## C++ 산술 연산자

* ### 연산 순서 : 우선 순위와 결합 방향

    * #### 우선 순위(precedence)

      * 하나의 피연산자에 하나 이상의 연산자가 걸렸을 때 어느 연산자를 먼저 적용할 것인지를 정하는 규칙이다.
      * 산술 연산자는 일반 대수학의 우선순위 규칙을 따른다.

    * #### 결합 순서

      * ```cpp
          float logs = 120 / 4 * 5;	// 150? 6?
        ```

      * 하나의 피연산자에 우선순위가 같은 두 연산자가 걸려 있을 경우에, **결합 방향이 왼쪽에서 오른쪽인지 또는 오른쪽에서 왼쪽인지를 나타내는 결합 방향 규칙을 적용**한다.

    

    * ### 데이터형 변환

      * 데이터형을 혼합하여 사용하면 데이터형의 불일치를 해결하기 위해 자동으로 데이터형 변환을 수행한다.

        * 특정 데이터형의 변수에 다른 데이터형의 값을 대입했을 때
        * 수식에 데이터형을 혼합하여 사용했을 때
        * 함수에 매개변수를 전달할 때

      * #### 대입 구문에서의 데이터형 변환

        * C++에서는 어떤 데이터형의 값을 다른 데이터형의 변수에 대입하는 것이 자유롭다.
        * 이때마다 그 값은 대입될 변수의 데이터형으로 변환된다.
        * 데이터 형 변환에 따라 일어날 수 있는 잠재적인 문제점들이 있다.
          * double → float와 같이 큰 부동 소수점형을 작은 부동 소수점형으로
            * 정밀도(유효 숫자)가 손실된다. 원래 값이 변환 데이터형의 범위를 벗어날 경우, 결과를 예측할 수 없다.
          * 부동 소수점형을 정수형으로
            * 소수부를 잃어버린다. 원래 값이 변환 데이터형의 범위를 벗어날 경우, 결과를 예측할 수 없다.
          * long → short와 같이 큰 정수형을 작은 정수형으로
            * 원래 값이 변환 데이터형의 범위를 벗어날 경우, 대개 하위 바이트들만 복사된다.

      * #### {}를 사용할 경우의 초기화 변환(C++11)

        * C++11은 중괄호를 이용하여 리스트를 초기화 할 수 있다.

        * 변수형이 대입된 값으로 표현할 수 없을 경우에는 narrowing을 허용하지 않는다.

          * narrowing : 초기화를 통해 발생하는 암시적인 데이터의 손실

        * ```cpp
          const int code = 66;
          int x = 66;
          char c1 {31325};	// narrowing, 허용 안됨
          char c2 = {66};		// char가 66을 유지할 수 있기 때문에 허용
          char c3 = {code};	// 상수형 int 변수 code가 66이기 때문에 위와 같은 이유로 허용
          char c4 = {x};		// x가 상수가 아니기 때문에 허용 안됨
          x = 31325;
          char c5 = x;		//초기화에 의해 허용됨
          ```

      * #### 수식에서의 데이터형 변환

        * 수식에 여러 가지 데이터형을 혼합하여 사용하는 경우에 C++는 두 종류의 자동 데이터형 변환을 수행한다.

        * ##### 데이터형이 나타날 때마다 자동으로 데이터형이 변환

          * 어떤 수식의 값을 평가할 때 bool, char, unsigned char, signed char, short형 값은 모두 int형으로 변환된다.

            * 이러한 변환을 **정수 승급(integral promotion)**이라 한다.

          * ```cpp
            short chickens = 20;
            short ducks = 35;
            short fowl = chickens + ducks;
            ```

            * C++ 프로그램은 chickens와 ducks의 값을 int형으로 각각 변환한 후에 더한다.
            * 그 더한 결과를 short형으로 다시 변환하여 short형 변수 fowl에 대입한다.
            * 이러한 이유는 일반적으로 **컴퓨터 내부에서 가장 자연스럽게 처리되는 데이터형은 int형이기 때문이다.**(컴퓨터가 int형에 대해 가장 빠르게 계산을 수행한다.)

        * ##### 수식에서 다른 데이터형과 혼합하여 사용되었을 때 데이터형이 변환

          * 하나의 연산에 두 가지 데이터형이 들어 있으면 작은 크기의 데이터형이 큰 크기의 데이터형으로 변환된다.
            * ex) 9.0을 5로 나눌 때, 9.0이 double형이기 때문에 나눗셈 연산을 수행하기 전에 int 형인 5가 double형으로 변환된다.
          * 일반적으로 컴파일러는 산술식에서 어떤 데이터형 변환이 이루어져야 하는지를 검사한다.
            * 부동소수점형의 경우, 한쪽 피연산자의 데이터형에 따라 상대편 피연산자의 데이터형을 변환한다.
            * 피연산자들이 정수형인 경우 정수 승급이 일어난다.
              * 양쪽 피연산자 모두 signed이거나, 모두 unsigned일 경우에는 두 피연산자의 상대적 크기에 따라 큰 쪽으로 변환한다.
              * 한쪽 피연산자가 signed이고, 다른 쪽이 unsigned일 경우에는 unsigned 피연산자가 signed 피연산자보다 상대적으로 크다면 unsigned 피연산자의 형으로 변환한다.
              * signed형이 unsigned형의 모든 값을 표현할 수 있다면, unsigned 피연산자가 signed형으로 변환된다.
