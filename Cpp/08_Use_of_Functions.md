# 함수의 활용

- ## C++ 인라인(inline) 함수

  - ### 일반적인 함수의 호출

    1. 프로그램이 **함수 호출 명령에 도달**했을 때, 그 함수 호출 명령 바로 다음에 있는 **명령의 주소를 메모리에 저장**한다.
    2. **스택에 매개변수를 복사**한 후, 그 함수가 시작되는 **메모리 위치로 점프**한다.
    3. **함수의 코드를 수행**하고, **함수의 리턴값을 레지스터에 복사**한 후, **메모리에 저장해 두었던 주소의 명령으로 돌아온다.**

    - 이렇게 앞뒤로 점프를 수행하고, 점프할 위치를 기억하려면 **함수를 사용하는 데 시간이 많이 걸린다.**

  

  - ### 인라인 함수

    - 컴파일된 **함수 코드**가 프로그램의 **다른 코드 안에 직접 삽입되어 있다.**

      - 컴파일러가 **함수 호출을 그에 대응하는 함수 코드로 대체**한다.

    - 함수를 그 자리에서 처리하므로, 그 **함수를 수행하기 위해** 프로그램이 **다른 주소로 점프했다가 돌아올 필요가 없다.**

      - 따라서 인라인 함수는 **일반 함수보다 약간 빠르게 수행된다.**

    - **메모리 사용 측면**에서는 인라인 함수가 **일반 함수보다 불리하다.**

      - **인라인 함수를 호출한 만큼** 프로그램은 **함수의 사본을** 프로그램의 코드 안에 **삽입해야 한다.**

    - 함수 코드 자체를 수행하는 데 걸리는 시간이, 함수 호출의 과정을 처리하는 데 걸리는 시간에 비해 매우 길다면, 전체적으로 절약되는 시간은 거의 없다.

      - 함수 코드를 수행하는 데 걸리는 시간이 **매우 짧을 경우**에만 **인라인 함수를 사용**하는 것이 일반 함수를 사용하는 것보다 **시간이 절약된다.**

    - 인라인 함수의 사용은 상대적으로 빠른 함수 호출 과정의 이점을 죽이는 것이다.

      - **빈번하게 호출되는 함수가 아니라면**, 절대적인 **시간 절약은 그다지 크다고 볼 수 없다.**

    - #### 인라인 함수 사용 법

      - **함수 선언 앞에** inline이라는 키워드를 붙인다.

      - **함수 정의 앞에** inline이라는 키워드를 붙인다.

      - 일반적인 관행은 원형을 생략하고, 원형이 놓일 자리에 함수 정의 전체를 놓는(함수 머리와 함수 코드)를 놓는 것이다.

      - **함수의 덩치가 크다**는 이유로 또는 **재귀 호출**이라는 이유로 인라인 함수를 만드는 것을 **거절하는 컴파일러도 있다.**

        - 컴파일러가 인라인 함수 기능을 구현하지 못할 수도 있다.

      - #### 인라인 함수 정의 예

        - ```cpp
          inline double square(double x) { return x * x; }
          ```

  

  - ### 인라인과 매크로

    - 인라인 기능은 C++에 새로 추가된 것이다.

    - C는 매크로 기능을 제공하기 위해 선행처리기 #define 구문을 사용한다.

    - **매크로는 인라인 코드를 소스적으로 구현한 형태이다.**

      - ```cpp
        #define SQUARE(X) X*X
        ```

      - 매크로는 함수에 매개변수를 넘기는 것이 아니라 **텍스트를 대체함으로써 동작한다.**

      - 여기서 X가 '매개변수'를 나타내는 기호 이름 역할을 수행한다.

        - a = SQUARE(5.0);은 a = 5.0*5.0;으로 대체된다.

          b = SQUARE(4.5 + 7.5);는 b = 4.5 + 7.5*4.5 + 7.5;로 대체된다.

          d = SQUARE(c++);는 d = c++*c++;로 대체된다.

        - 이 경우, 첫 번째 예제만 제대로 작동한다. 나머지 두 예제가 제대로 작동하려면 괄호를 사용해야 한다.

          - ```cpp
            #define SQUARE(X) {(X)*(X)}
            ```

        - 그래도 매크로가 값을 전달하지 않는다는 문제는 여전히 남는다.

        - 괄호를 사용하여 새롭게 정의했어도 SQUARE(c++)는 c를 두 번이나 증가시킨다.

      - 그러나 인라인 함수를 사용하면 제곱을 원하는 값을 함수에 전달한 후, c를 한 번만 증가시킨다.

      - 경우에 따라 **C의 매크로**를 **C++의 인라인 함수로 변환**하는 것을 고려해야 한다.



- ## 참조 변수

  - 참조(reference)는 미리 정의된 어떤 변수의 실제 이름 대신 쓸 수 있는 **대용 이름이다.**

  - ### 참조 변수의 생성

    - & 기호를 사용하여 참조 선언을 할 수 있다.

      - ```cpp
        int rats;
        int & rodents = rats;	//rodents를 rats의 대용 이름으로 만든다
        ```

    - #### 참조와 포인터의 차이

      - ```cpp
        int rats = 101;
        int & rodents = rats;	//rodents는 참조이다.
        int * prats = &rats;	//prats는 포인터이다.
        ```

        - rodents나 *prats를 rats와 서로 바꾸어 사용할 수 있다.
        - &rodents나 prats를 &rats와 서로 바꾸어 사용할 수 있다.

      - ```cpp
        int rat;
        int & rodent;
        rodent = rat;	//이렇게 사용할 수 없다.
        ```

        - 참조를 **선언할 때는** 참조 변수를 함께 **초기화** 해야 한다.
        - 참조를 먼저 선언하고 나중에 값을 지정할 수는 없다.

      - ```cpp
        int & rodents = rats;
        int * const pr = &rats;
        ```

        - 참조는 const 포인터와 상당히 비슷하다.
        - 참조를 만들 때 반드시 초기화를 해야 하고, 어떤 특정 변수에 연결되면 그것을 고수해야 한다.
    
  - ### 함수 매개변수로서의 참조

    - 함수에서 사용하는 변수의 이름을 그 함수를 호출한 프로그램(호출 함수)에 있는 변수의 대용 이름으로 만든다.
      
      - 이러한 방식으로 매개변수를 전달하는 것을 **참조로 전달**이라 한다.
      
    - 참조로 전달하면 **피호출 함수가 호출 함수의 변수를 사용**할 수 있다.

    - 피호출 함수는 호출 함수가 건네주는 값의 **복사본을 대상으로 작업**한다.

    - ```cpp
      swapr(int & a, int & b);	//참조를 이용한 함수
      swapp(int * p, int * q);	//포인터를 이용한 함수
      swapv(int a, int b);	//값으로 전달하는 함수
      
      swapr(wallet1, wallet2);	//변수를 전달
      swapp(&wallet1, &wallet2);	//변수의 주소를 전달
      swapv(wallet1, wallet2);	//변수의 값을 전달
      ```

      - swapr() : **참조를 이용한 함수**로 a와 b는 각각 wallet1과 wallet2의 대용 이름으로 작동하기 때문에, a와 b를 교환하면 wallet1과 wallet2가 교환된다.
      - swapp() : **포인터를 이용한 함수**로 포인터를 이용하는 경우에는 함수가 p와 q를 사용할 때마다 내용 **참조 연산자인 *를 사용해야 한다.**
      - swapv() : **값으로 전달하는 함수**로 변수 a, b는 단순히 wallet1, wallet2를 각각 복사한 값이므로, a와 b를 교환해도 **원본**인 wallet과 wallet2는 **영향을 받지 않는다.**
    
  - ### 참조의 특성
  
    - 함수에 전달하는 정보를 그 함수가 변경하지 않고 사용만 하도록 하려면, 상수 참조를 사용해야 한다.
  
      - ```cpp
        double refcube(const double &ra);
        ```
  
      - 컴파일러가 ra의 값을 변경하려고 시도하는 코드를 발견했을 때 에러 메시지를 내보낸다.
  
    - #### 임시 변수, 참조 매개변수, const
  
      - C++는 실제 매개변수와 참조 매개변수가 일치하지 않을 때 임시 변수를 생성할 수 있다.
        - 최근 C++는 매개변수가 const 참조일 경우에만 이것을 허용한다.
      - 임시 변수가 생성되는 상황
        - 실제 매개변수가 올바른 데이터형이지만 lvalue가 아닐 때
          - lvalue : 참조가 가능한 데이터 객체이다.
            - 변수, 배열의 원소, 구조체의 멤버, 참조 또는 역참조 포인터
            - 일반 상수와 여러 개의 항으로 이루어진 표현식은 아니다.
        - 실제 매개변수가 잘못된 데이터형이지만 올바른 데이터형으로 변환할 수 있을 때
  
    - #### 정리
  
      - 함수 호출 실제 매개변수가 lvalue가 아니거나, 그에 대응하는 const 참조 매개변수의 데이터형과 일치하지 않을 때, C++는 정확한 데이터형의 익명 변수를 생성하고, 그 익명 변수에 함수 호출 실제 매개변수의 값을 대입하고, 그 익명 변수를 참조하게 한다.
  
    - #### 가능한 const를 사용하자
  
      - const를 사용하면, 실수로 데이터 변경을 일으키는 프로그래밍 에러를 막을 수 있다.
      - 원형에 const를 사용하면, 함수가 const와 const가 아닌 실제 매개변수를 모두 처리할 수 있지만, 원형에 const를 생략한 함수는 const가 아닌 데이터만 처리할 수 있다.
      - const 참조를 사용하면, 함수가 자신의 필요에 따라 임시 변수를 생성하여 사용할 수 있다.
