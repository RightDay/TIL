* ## 포인터와 메모리 해제

  * 포인터는 값 자체가 아니라 **값의 주소를 저장하는 변수**이다.

  * 주소 연산자(&)를 변수 앞에 붙이면 그 **변수의 주소를 알아낼 수 있다.**

    * home이 변수면 &home은 그 변수의 주소이다.

    * ```cpp
      int donuts = 6;
      double cups = 4.5;
      
      cout << "donuts의 값 = " << donuts;
      cout << ", donuts의 주소 = " << &donuts << endl;
      
      cout << "cups의 값 = " << cups;
      cout << ", cups의 주소 = " << &cups << endl;
      ```

      * donuts = 6, donuts의 주소 = 0012FF7C

        cups의 값 = 4.5, cups의 주소 = 0012FF74

    * 16진수 표기가 메모리를 나타내는 가장 일반적인 방법이기 때문에 **주소값을 출력할 때 16진수 표기를 사용한다.**

    * 이 C++에서는 cups를 donuts보다 낮은 메모리 위치에 저장하였다.

      * 여기서 두 주소 간의 차이는 0012FF7C - 0012FF74이므로 8바이트이다.
      * cups가 8바이트를 사용하는 double형이기 때문이다.

    * C++ 시스템 마다 출력하는 주소가 다르다.

  

  
  * ### 포인터와 C++의 철학
  
    * 객체 지향 프로그래밍은 컴파일 시간이 아닌 **실행 시간에 어떤 결정을 내린다**는 점을 강조한다.
    * ex) 객체 지향 프로그래밍에서는 배열의 크기를 실행 시간에 결정할 수 있다.
      * 이것이 가능하려면, 프로그램이 실행되는 동안에 배열 또는 그와 동등한 것을 생성할 수 있어야 한다.
    * C++에서는 **new라는 키워드를 사용하여 원하는 만큼의 메모리를 대입**하고, 이렇게 **대입된 메모리의 위치를 포인터를 사용하여 추적**할 수 있다.
    
  * 포인터의 이름이 **주소**를 나타낸다.
  
    * **간접값(indirect value) 연산자** 또는 **간접 참조(dereferencing) 연산자**라고 부르는 *를 포인터 이름 앞에 붙이면 그 **주소에 저장되어 있는 값**이 된다.
  
    * ```cpp
      int updates = 6;	//int형 변수를 선언
      int * p_updates;	//int형을 지시하는 포인터를 선언
      
      p_updates = &updates;	//int형의 주소를 포인터에 대입
      
      //값을 두 가지 방법으로 표현
      cout << "값 : updates = " << updates;
      cout << ", *p_updates = " << *p_updates << endl;
      
      //주소를 두 가지 방법으로 표현
      cout << "주소 : &updates = " << &updates;
      cout << ", p_updates = " << p_updates << endl;
      
      //포인터를 사용하여 값을 변경
      *p_updates = *p_updates + 1;
    cout << "변경된 updates = " << updates << endl;
      return 0;
    ```
  
    * 실행 결과
  
      값 : updates = 6, *p_updates = 6
  
        주소 : &updates = 0012FF7C, p_updates = 0012FF7C
  
      변경된 updates = 7
  

  
  
  * ### 포인터의 선언과 초기화

    * 포인터를 선언할 때에는 **포인터가 지시하는 데이터형이 무엇인지 서술해야 한다.**

      * ```cpp
      int * p_updates;
        ```

      * p_updates는 포인터(주소)이고, *p_updates는 포인터가 아닌 int형 변수이다.
  
    * 서로 다른 크기의 데이터형을 지시하는 포인터이더라도, **두 변수의 크기는 같다.**
  
  * **선언 구문을 사용**하여 포인터를 초기화할 수 있다.
  
    * ```cpp
        int higgens = 5;
      int * pt = &higgens;
        ```

      * 포인터에 의해 지시되는 값인 *pt가 초기화되는 것이 아니라, 포인터인 pt가 &higgens 값으로 초기화된다.
  
  
  

  * ### 포인터의 위험

    * C++에서 포인터를 생성하면 컴퓨터는 **주소를 저장하기 위한 메모리를 대입**한다.

      * ``` cpp
      long * fellow;	//long형을 지시하는 포인터 생성
        *fellow = 223323;	//어딘지 알 수 없는 곳에 값을 저장
      ```
  
    * fellow 포인터를 초기화하지 않았기 때문에 fellow의 엉뚱한 주소에 값이 저장된다.
  
  * 포인터를 사용할 때에는 **간접 참조 연산자(*)를 사용하기 전에** 반드시 **포인터를 적절한 주소로 초기화**시켜 주어야 한다.
  

  
  
  * ### 포인터와 수
  
  * 컴퓨터가 주소를 정수로 다루고 있지만, **포인터는 정수형이 아니다.**
  
    * 포인터는 정수와 달리 **위치를 나타낸다.**
  
      * 수행할 수 있는 **연산의 관점에서** 포인터와 정수형은 서로 다르다.
  
      * 포인터에 정수를 대입할 수 없다.
  
        * ```cpp
        int * pt;
          pt = 0xB8000000;	//데이터형 불일치
        ```
  
    * 어떤 수를 주소로 사용하려면 **데이터형 변환자를 사용하여 적당한 주소형으로 변환**해 주어야 한다.
  
        * ```cpp
          int * pt;
        pt = (int *) 0xB8000000;	//데이터형 일치
          ```
  
  
  
  * ### new를 사용한 메모리 대입
  
    * C++는 **new 연산자를 사용하여 메모리에 접근**한다.
  
    * 어떤 데이터형의 메모리를 원하는지 new 연산자에게 알려 주면, new 연산자가 그에 알맞은 크기에 메모리 블록을 찾아내고, 그 블록의 주소를 리턴한다.
  
      * ```cpp
        int * pn = new int;
        ```
  
        * new int 부분은 int형 데이터를 저장할 새로운 메모리가 필요하다고 프로그램에 알린다.
        * new 연산자는 뒤따르는 데이터형을 보고, 몇 바이트가 필요한지를 파악한다.
        * 그 이후, 적당한 메모리를 찾아 필요한 만큼 블록을 대입하고, 그 주소를 리턴한다.
        * 이제 리턴되는 주소는 int형을 지시하는 포인터로 선언되어 있는 pn에 대입된다.
        
          * 여기서 pn은 주소이고, *pn은 그 주소에 저장되는 값이다.
        
      * ```cpp
        int higgens;
        int * pt = &higgens;
        ```
        
        * 두 방식 모두 int형의 주소가 포인터(pn과 pt)에 대입된다.
          * 첫 번째 방식에서는 포인터 pn이 int형 값에 접근할 수 있는 유일한 통로이다.
            * pn이 지시하는 메모리는 이름이 없지만 **데이터 객체(data object)**를 지시하고 있기 때문에 메모리 접근이 가능하다.
            * 데이터 객체는 어떤 **데이터를 저장하기 위해 대입된 메모리 블록**을 의미한다.
          * 두 번째 방식에서는 pt를 통하지 않고 변수 이름 higgens를 통하여 int형 값에 접근할 수 있다.
        
      * ```cpp
          int * pt = new int;	//int형을 저장할 메모리를 대입
          *pt = 1001;
          
          double * pd = new double;	//double형을 저장할 메모리를 대입
          *pd = 10000001.0;
          
          cout << "pt의 크기 : " << sizeof(pt);
          cout << " *pt의 크기 : " << sizeof(*pt) << endl;
          
          cout << "pd의 크기 : " << sizeof(pd);
          cout << " *pd의 크기 : " << sizeof(*pd) << endl;
          ```
          
          * 실행 결과 :
          
            pt의 크기 : 4 *pt의 크기 : 4
          
            pd의 크기 : 4 *pd의 크기 : 8
          
          * pt와 pd는 단순한 주소이기 때문에 모두 4바이트이다.
            
          * *pt는 4바이트를 차지하는 int형 값, *pd는 8바이트를 차지하는 double형 값이다.
  
  
  
  * ### 메모리 부족
  
    * 컴퓨터에 사용할 수 있는 메모리가 부족하여 **new의 메모리 대입 요청을 허용할 수 없는 상황일 경우 new는 0값을 리턴**한다.
      * C++에서는 값이 0인 포인터를 **널 포인터(null pointer)**라고 부른다.
      * 널 포인터는 어떠한 데이터도 지시하지 않는다.
      * 널포인터는 사용할 수 있는 **포인터를 리턴해야 하는 연산자**나 **함수에서 오류를 나타낼 때** 흔히 사용된다.
    * 메모리를 대입하지 못했을 때 널 포인터를 리턴하는 것 외에,  **bad_alloc 예외를 리턴**할 수 있다.
  
  
  
  * ### delete를 사용한 메모리 해제
  
    * delete 연산자를 통해 **메모리를 다시 메모리풀로 환수(해제)**할 수 있다.
  
    * 환수 또는 해제한 메모리는 **프로그램의 다른 부분이 다시 사용**할 수 있다.
  
    * delete는 new로 대입한 **메모리 블록을 지시하는 포인터와 함께 사용**한다.
  
      * ```cpp
        int * ps = new int;	//new로 메모리 대입
        ...	//메모리 사용
        delete ps;	//delete로 메모리 해제
        ```
  
      * ps가 지시하는 메모리가 해제되나, ps 자체가 없어지는 것은 아니다.
  
      * ps는 새로 대입한 메모리를 지시하는 데 다시 사용할 수 있다.
  
    * **new를 사용한 후에는 반드시 나중에 delete를 사용해야 한다.**
  
      * 그렇지 않으면 대입은 되었지만 더 이상 사용이 되지 않는 **메모리 누수가 발생**할 수 있다.
      * 메모리 누수의 규모가 너무 커지면 메모리를 많이 요구하는 프로그램이 먹통이 될 수도 있다.
  
    * **이미 해제한 메모리 블록을 다시 해제하려고 시도해선 안 된다.**
  
      * C++ 표준은 이러한 시도를 했을 때의 결과는 정의되지 않는다고 말한다.
  
    * **보통의 변수를 선언하여 대입한 메모리는 delete로 해제할 수 없다.**
  
      * ```cpp
        int * ps = new int;	//맞다.
        delete ps;	//맞다.
        delete ps;	//이미 해제한 메모리 블록을 다시 해제하려고 시도하므로 틀리다.
        int jugs = 5;	//맞다.
        int * pi = &jugs;	//맞다.
        delete pi;	//new로 대입한 메모리가 아니므로 틀리다.
        ```
  
    * **delete는 new로 대입한 메모리를 해제할 때에만 사용해야 한다.**
  
      * new가 사용한 것과 동일한 포인터 변수에 사용해야 한다는 뜻이 아니라, **동일한 주소에 사용해야 한다**는 뜻이다.
  
        * ```cpp
          int * ps = new int;	//메모리를 대입하고 첫 번째 포인터로 지시
          int * pq = ps;	//같은 메모리를 지시하는 두 번째 포인터 설정
          delete pq;	//두 번째 포인터에 delete 사용
          ```
  
        * 일반적으로, **같은 메모리 블록을 지시하는 두 개의 포인터를 만드는 것**은, 실수로 **같은 메모리 블록을 두 번 삭제하려고 시도할 가능성이 있기 때문에 좋지 않다.**
  
      * delete를 **널 포인터에 사용하는 것은 안전**하다.
  
  
  
  * ### new와 delete를 사용할 때 지켜야할 규칙
  
    * new로 대입되지 않은 메모리는 delete로 해제하지 않는다.
    * 같은 메모리 블록을 연달아 두 번 delete로 해제하지 않는다.
    * new []로 메모리를 대입한 경우에는 delete []로 해제한다.
    * new를 대괄호 없이 사용했으면 delete도 대괄호 없이 사용한다.
    * 널 포인터에는 delete를 사용하는 것이 안전하다.(아무 일도 일어나지 않는다.)
  
  
  
  * ### new를 사용한 동적 배열의 생성
  
    * 배열, 문자열, 구조체와 같은 **큰 데이터를 다룰 때에는 new를 사용하는 것이 효율적**이다.
  
    * 정적 바인딩(static binding) : 프로그램이 **컴파일될 때 메모리가 대입**된다.
  
      * ex) 프로그램을 작성할 때 배열의 크기를 정한다.
  
    * 동적 바인딩(dynamic binding) : 프로그램이 **실행되는 동안에 크기를 결정**한다.
  
      * ex) 프로그램이 실행되는 동안에 배열의 크기를 정할 수 있다.
  
    * 동적 배열(dynamic array) : 동적 바인딩으로 생성된 배열
  
    * #### 동적 배열의 생성
  
      * 배열 원소의 **데이터형과 개수**를 new에 알려 준다.
  
        * ```cpp
          int * psome = new int [10];	//10개의 int형 값을 저장할 블록을 대입
          ```
  
        * new 연산자는 블록의 **첫 번째 원소의 주소를 리턴**한다.
  
        * 이 예제에서는 포인터 psome에 주소가 대입된다.
  
      * **new를 사용하여 생성된 메모리 블록**은, 프로그램이 사용을 끝낸 후에 반드시 **delete로 해제**해 주어야 한다.
  
        * ```cpp
          delete [] psome;	//동적 배열을 해제
          ```
  
        * 대괄호 []가 사용되었기 때문에 포인터가 지시하는 첫 번째 원소가 해제되는 것이 아니라 **배열 전체가 해제**된다.
  
        * 대괄호는 **delete와 포인터 사이에 사용**된다.
  
        * new를 대괄호 없이 사용했으면 delete도 대괄호 없이 사용해야 한다.(반대도 마찬가지)
  
          * ```cpp
            int * pt = new int;
            short * ps = new short [500];
            delete [] pt;	//결과 미정의, 사용 불가
            delete ps;	//결과 미정의, 사용 불가
            ```
  
      * 동적배열 포인터는 그 **블록의 첫 번째 원소인 데이터형을 지시하는 포인터**이다.
  
        * 컴파일러는 그 **포인터가 블록의 첫 번째 원소를 지시하고 있다는 사실을 알지 못한다.**
        * 그렇기 때문에 **원소의 개수를 알 수 있도록 프로그램을 작성**해야 한다.
  
      * delete [] 연산자로 메모리를 바르게 해제할 수 있도록 **내부적으로 프로그램은 대입되는 메모리의 양을 추적**한다.
  
        * 그러나 프로그래머는 그 정보를 사용할 수 없다.
        * ex) 동적 배열에 대입된 바이트 수를 알아보기 위해 **sizeof 연산자를 사용할 수 없다.**
  
      * 동적 배열을 위한 메모리를 대입하고, 그 배열의 시작 주소를 포인터로 대입하는 일반 형식은 다음과 같다.
  
        * ```cpp
          type_name * pointer_name = new type_name [num_elements];
          ```
  
        * type_name형의 원소를 num_elements만큼 저장할 수 있는 메모리 블록에 대입한다.
  
        * 블록의 첫 번째 원소의 주소를 pointer_name에 대입한다.
  
        * pointer_name을 배열 이름처럼 사용할 수 있다.
  
    * #### 동적 배열의 사용
  
      * ```cpp
        int * psome = new int [10];	//10개의 int형 값을 위한 블록을 대입
        ```
  
        * psome이 배열의 첫 번째 원소를 지시하는 포인터이기 때문에, *psome은 첫 번째 원소의 값이 된다.
    
        * 나머지 원소들은 psome[0], psome[1]과 같이 나타낼 수 있다.
    * 이는 C와 C++가 배열을 구현할 때 내부적으로는 포인터를 사용하기 때문이다.
        
  * ```cpp
        double * p3 = new double [3];
        
        p3[0] = 0.2;
        pe[1] = 0.5;
        p3[2] = 0.8;
        
        cout << "p3[1]은 " << p3[1] << "입니다.\n";
        
        p3 = p3 + 1;	//포인터 증가
        
        cout << "이제는 pe[0]이 " << p3[0] << "이고, ";
        cout << "p3[1]이 " << p3[1] << "입니다.\n";
        
        p3 = p3 - 1;	//다시 시작 위치를 지시
      
    delete [] p3;	//배열 메모리를 해제
        
        return 0;
        ```
    
      * ```cpp
    p3 = p3 + 1;	//p3이 포인터면 맞고, 배열 이름이면 틀리다.
        ```
      
        * 연속적으로 저장되어 있는 int형 값들의 실제 주소는 대개 2바이트나 4바이트씩 차이가 난다. 
        
        * p3에 1을 더하면 실제로는 그 다음 차례 원소의 주소를 지시하게 된다.





* ## 포인터, 배열, 포인터 연산

  * **포인터 변수에 1을 더하면** 값이 그 포인터가 지시하는 **데이터형의 바이트 수만큼 증가**한다.

    * ex) double형을 지시하는 포인터에 1을 더하는 것은 포인터의 수치 값에 8을 더하는 것과 같다.

  * C++는 **배열 이름을 그 배열의 첫 번째 원소의 주소로 해석**한다.

    * ```cpp
      double wages[3] = {10.0, 20.0, 30.0};
      double * pw = wages;
      ```

    * ```cpp
      wages = &wages[0] = 배열의 첫 번째 원소의 주소
      ```

    * ```cpp
      wages[i]는 *(wages + i)와 같다.
      ```

    * ```cpp
      pointername = pointername + 1;	//맞다.
      arrayname = arrayname + 1;	//틀리다.
      ```

  * **배열 이름에 sizeof 연산자**를 적용하면 **배열의 크기**가 얻어진다.

  * **포인터에 sizeof 연산자**를 적용하면 비록 그 포인터가 배열을 지시하고 있더라도 **포인터의 크기**가 얻어진다.

  * ### 배열의 주소

    * 배열의 이름은 **첫 번째 원소의 주소**를 가리킨다.

      * 반면에, 그 주소를 통해 전체 배열의 주소를 얻어낸다.

    * ```cpp
      short tell[10];	//tell은 20바이트 배열이다.
      cout << tell << endl;	//&tell[0]을 출력한다.
      cout << &tell << endl;	//전체 배열의 주소를 출력한다.
      ```

      * &tell[0]과 tell은 2바이트 메모리 블록으로 동일한 값을 갖는다.
      * &tell은 20바이트 메모리 블록의 주소이다.
      * tell + 1은 주소값에 2를 더하게 되고 &tell + 1은 주소값에 20을 더하게 된다.

    * ```cpp
      short (*pas)[20] = &tell;	//pas는 20바이트 배열을 가리킨다.
      ```

      * 만약, 괄호가 빠졌을 경우에는 pas는 short를 가리키는 20개의 포인터의 배열이 된다.
      * pas의 형은 short (*)[20]이다.
      * pas가 &tell으로 설정되기 때문에 *pas는 tell과 동일하고, (\*pas)[0]는 tell 배열의 첫 번째 원소가 된다.
    
  

  
* ### 포인터에 관한 요약
  
    * #### 포인터 선언
  
      * ```cpp
        typeName * pointerName;
        
        //ex
      double * pn;	//pn은 double형 값을 지시할 수 있다.
        char * pc;	//pc는 char형 값을 지시할 수 있다.
      ```
  
  * #### 포인터에 값 대입
  
    * 포인터에는 **메모리 주소를 대입**해야 한다.
  
    * 변수 이름에 **& 연산자**를 적용하면 이름이 있는 **메모리의 주소**를 얻을 수 있다.
  
      * **new 연산자**는 **이름이 없는 메모리의 주소**를 리턴한다.
  
      * ```cpp
        double * pn;	//pn은 double형 값을 지시할 수 있다.
        double * pa;	//pa는 double형 값을 지시할 수 있다.
        char * pc;	//pc는 char형 값을 지시할 수 있다.
        
        double bubble = 3.2;
        pn = &bubble;	//bubble의 주소를 pn에 대입한다.
      pc = new Char;	//새로 대입된 char형 메모리의 주소를 pc에 대입한다.
        pa = new double[30];	//double형 30개 배열의 주소를 pa에 대입한다.
      ```
  
  * #### 포인터의 간접 참조
  
    * **포인터가 지시하는 주소에 저장되어 있는 값을 참조**한다는 뜻이다.
  
      * 간접 값(indirect value) 연산자 또는 간접 참조(dereferencing) 연산자라 부르는 * 연산자를 포인터에 적용하면, 그 **포인터가 지시하는 주소에 저장되어 있는 값을 참조**할 수 있다.
  
      * ```cpp
        double bubble = 3.2;
        pn = &bubble;	//bubble의 주소를 pn에 대입한다.
      cout << *pn;	//pn이 지시하는 bubble의 값을 출력한다.
        *pn = 4.5;	//주소가 pn인 메모리 위치에 값 4.5를 대입한다.
      ```
  
    * 배열 표기는 **포인터를 사용하여 값을 간접적으로 참조**하는 또 다른 방법이다.
  
      * ex) pn[0]은 *pn과 같다.
  
    * 포인터를 **적당한 주소로 초기화하지 않은 상태에서 간접 참조에 사용하면 안 된다.**
  
    * #### 포인터와 참조값의 구별
  
      * ```cpp
      int * pt = new int;	//포인터 pt에 주소를 대입한다.
        *pt = 5;	//그 주소에 값 5를 저장한다.
      ```
  
      * pt가 int형을 지시하는 포인터라면, *pt는 int형을 지시하는 포인터가 아니라 int형 변수와 완전히 동등한 것이다.
  
  * #### 배열 이름
  
      * 대부분의 경우에 C++는 **배열 이름을 그 배열의 첫 번째 원소의 주소와 동등한 것**으로 취급한다.
  
      * ```cpp
          int tacos[10];	//tacos는 &tacos[0]과 동등하다.
        ```
  
      * 배열 이름에 sizeof 연산자를 적용할 때에는 배열의 첫 번째 원소의 크기가 아니라 배열의 전체 크기가 바이트 단위로 리턴된다.
  
  * #### 포인터 연산
  
    * C++에서는 포인터에 정수를 더할 수 있다.
  
    * 포인터에 1을 더하는 것은 원래의 주소 값에 그 포인터가 지시하는 **데이터 객체의 크기를 더하는 것**과 같다.
  
        * 반대로 포인터에서 정수를 뺄 수 있다.
  
      * ```cpp
        int tacos[10] = {5, 2, 8, 4, 1, 2, 2, 4, 6, 8};
        int * pt = tacos;	//pt와 tacos가 주소 3000이라고 가정할 때
        pt = pt + 1;	//int형이 4바이트이면, 이제 pt는 3004이다.
        int * pe = &tacos[9];	//int형이 4바이트이면, pe는 3036이다.
      pe = pe - 1;	//이제 pe는 tacos[8]의 주소인 3032이다.
        int diff = pe - pt;	//diff는 tacos[8]과 tacos[1]의 간격 7이다.
        ```

        * 두 포인터의 차를 구하는 연산은 두 포인터가 같은 배열을 지시하고 있을 때에만 의미 있다.
        * 연산 결과는 **두 원소 사이의 간격을 나타내는 정수**이다.
  
  * #### 배열의 동적 바인딩과 정적 바인딩
  
      * **정적 바인딩으로 배열 크기가 고정된 배열**을 생성할 때에는 **배열 선언 구문을 사용**할 수 있다.
  
      * ```cpp
          int tacos[10];	//정적 바인딩, 컴파일 시간에 크기가 정해진다.
        ```
  
    * 동적 바인딩으로 **동적 배열을 생성할 때에는 new 연산자를 사용**한다.
  
    * 동적 바인딩에서는 프로그램이 실행될 때 **배열의 크기가 정해지고 메모리를 대입**받는다.
  
      * 배열을 사용하는 작업이 끝나면 **delete[]로 메모리를 해제**해야 한다.
  
        * ```cpp
          int size;
          cin >> size;
          int * pz = new int[size];	//실행 시간에 크기가 정해지는 동적 바인딩
        ...
          delete [] pz;	//작업이 끝나면 메모리를 해제한다.
        ```
  
  * #### 배열 표기와 포인터 표기
  
      * 대괄호를 사용하는 배열 표기는 **포인터를 사용하는 간접 참조와 의미상 동등**하다.
  
        * tacos[0]은 *tacos와 동등하다.
        * 이들은 tacos 주소에 있는 값을 의미한다.
        * tacos[3]은 *(tacos + 3)과 동등하다.
        * 이들은 tacos + 3 주소에 있는 값을 의미한다.
  
      * 이와 같은 동등성은 배열 이름과 포인터 변수 둘 다에 해당된다.
  
      * ```cpp
        int * pt = new int[10];	//pt는 10개의 int형을 가진 블록을 지시한다.
        *pt = 5;	//첫 번째(0번) 원소에 5를 대입한다.
        pt[0] = 6;	//첫 번째(0번) 원소에 6을 다시 대입한다.
        pt[9] = 44;	//열 번째(9번) 원소에 44를 대입한다.
        int coats[10];
        *(coats + 4) = 12;	//coats[4]에 12를 대입한다.
      ```
  

  
  * ### 포인터와 문자열
  
  * cout을 비롯한 대부분의 C++ 표현에서, char형의 배열 이름, char형을 지시하는 포인터, 큰따옴표로 둘러싸인 **문자열 상수는 모두 문자열의 첫 번째 문자의 주소로 해석**된다.
  
    * ```cpp
      const char * bird = "wren";	//bird에 문자열의 주소가 들어 있다.
    ```
  
      * "wren"은 실제로는 그 문자열의 주소를 나타낸다.
      * 이 구문은 "wren"의 주소를 포인터 bird에 대입한다.
      * 일반적으로, 컴파일러는 프로그램의 소스 코드에 나타나는 모든 문자열을 메모리의 어떤 영역에 저장하고, 각각의 문자열과 주소를 서로 연결한다.
  
  * ```cpp
      const << "A concerned " << bird << " speaks\n";
    ```
  
    * 문자열에 const를 사용하면 포인터를 사용하여 그 문자열에 접근할 수 있지만 그 문자열을 변경할 수는 없다.
  
  * 프로그램 안으로 문자열을 읽어들일 때에는 언제나 **미리 대입된 메모리의 주소를 사용**해야 한다.
  
      * 이 주소는 **배열 이름 형식**일 수도 있고, **new를 사용하여 초기화한 포인터 형식**일 수도 있다.
  
  * #### cstring 라이브러리의 사용
  
      * ```cpp
        ps = new char[strlen(animal) + 1];	//새 저장 공간을 확보한다.
      ```
  
        1. strlen() 함수를 사용하여 **문자열의 길이**를 알아낸다.
        2. **널 문자를 저장**하기 위해 그 **길이에 1을 더한다.**
      3. **new를 사용**하여 문자열을 저장하는 데 꼭 **필요한 만큼의 메모리를 대입**한다.
  
      * ```cpp
        strcpy(ps, animal);	//문자열을 새로 대입받은 공간으로 복사한다.
      ```
  
        * strcpy() 함수는 두 개의 매개변수를 가진다.
        * 첫 번째 매개 변수는 문자열이 **복사될** 도착지 주소이고, 두 번째 매개변수는 **복사할** 문자열이 있는 주소이다.
        * 포인터 ps에 문자열 배열 animal을 대입하는 것은 단지 ps에 저장된 주소만 변경하기 때문에 새로 대입한 메모리에 문자열을 복사하지 못한다.
  
    * ```cpp
    char food[20] = "carrots";	//초기화
      strcpy(food, "flan");	//그 밖의 경우
    
      strcpy(food, "a picnic basket filled with many goodies");
      ```

      * 위의 strcpy() 함수는 문제가 있다.

        * food 배열의 크기가 저장하려는 문자열보다 작기 때문이다.
      * 이러한 경우에, **strcpy() 함수**는 **배열의 크기를 벗어나는 나머지 부분을 배열에 바로 이어서 저장하기 때문에, 프로그램이 사용하고 있는 다른 내용을 덮어 버릴 수 있다.**
  
    * 이러한 문제를 예방하려면 **strcpy() 대신에 strncpy()를 사용**해야 한다.
  
        * strncpy()는 **세 번째 매개변수를 사용하여 복사할 최대 문자 수를 저장**한다.
  
        * 이 함수는 문자열을 끝까지 복사하기 전에 저장 공간이 바닥이 나는 경우에 **널 문자를 추가하지 않는다.**

        * ```cpp
          strncpy(food, "a picnic basket filled with many goodies", 19);
        food[19] = '\0'
          ```

          * 19개의 영문자를 배열에 복사하고 마지막 원소를 널 문자로 설정한다.
        * 문자열이 19개의 영문자보다 짧을 경우에 strncpy() 함수는 문자열의 실제 끝을 표시하기 위해 **문자열 바로 뒤에 널 문자를 추가**한다.
  
  
  
* ### new를 사용한 동적 구조체의 생성
  
  * #### new를 사용한 구조체 생성
  
    * ```cpp
        inflatable * ps = new inflatable;
      ```
  
      * inflatable형의 구조체를 저장할 수 있는 크기의 새로운 메모리 블록을 확보하고, 그것의 주소를 포인터 ps에 대입한다.
  
  * #### 구조체의 멤버 접근
  
    * 동적 구조체에는 도트(.) 멤버 연산자를 사용할 수 없다.
  
      * 동적 구조체에는 이름이 없기 때문이다.
  
      * C++는 이를 해결하기 위해 화살표 멤버 연산자(->)를 제공한다.
  
    * 간접 참조 연산자(\*) 연산자를 사용하여 접근할 수 있다.
  
      * ps가 구조체를 지시하는 포인터일 때, *ps는 그 포인터가 지시하는 값, 즉 그 구조체가 된다.
  
      * ```cpp
          (*ps).price;	//구조체의 price 멤버 접근
        ```
  
        * 이 경우에는 C++의 연산자 우선순위 규칙 때문에 반드식 괄호를 사용해야 한다.
  
      * 도트 멤버 연산자는 구조체 이름과 함께 사용하고, 화살표 멤버 연산자는 구조체를 지시하는 포인터와 함께 사용한다.
  
    * #### new와 delete를 사용하는 예제
  
      * ```cpp
        char * getname(void);
        int main()
        {
            char * name;	//포인터를 만들지만 메모리는 대입하지 않는다.
            
            name = getname();	//문자열의 주소를 name에 대입한다.
            cout << (int *) name << ": " << name << "\n";
            delete [] name;	//메모리를 해제한다.
            
            name = getname();	//해제한 메모리를 다시 사용한다.
            cout << (int *) << ": " << name << "\n";
            delete [] name;	//메모리를 다시 해제한다.
            return 0;
        }
        
        char * getname()	//새 문자열을 가리키는 포인터를 리턴한다.
        {
            char temp[80];	//임시 배열
            cout << "이름을 입력하십시오: ";
          cin >> temp;
            char * pn = new char[strlen(temp) + 1];
          strcpy(pn, temp);	//문자열을 더 작은 공간으로 복사한다.
            
            return pn;	//함수가 종료될 때 임시 배열인 temp는 소멸한다.
        }
        ```
  
      * #### getname() 함수
  
      * getname() 함수는 cin을 통해 한 단어로 된 문자열을 입력받아 temp라는 임시 배열에 저장한다.
        * 문자열을 정식으로 저장할 새로운 메모리 블록을 **new를 사용하여 대입**한다.
        * 문자열을 저장하려면 널 문자까지 포함하여 strlen(temp) + 1 바이트가 필요하다. (이 값이 new에 전달 된다.)
        * 적당한 크기의 새로운 메모리 블록이 확보되면, 이 함수는 표준 라이브러리 함수인 **strcpy()를 사용**하여 temp에 들어 있는 **문자열을 새로운 메모리 블록으로 복사**한다.
      * getname() 함수는 문자열이 확보한 메모리 블록의 크기에 맞는지 검사하지 않는다.
          * 그러나 **new를 사용한 적당한 바이트 수를 대입**한 것이므로 아무런 문제가 없다.
      * 마지막으로 복사된 문자열이 저장되어 있는 메모리 블록의 주소 pn을 리턴한다.
  
* ### 자동 공간, 정적 공간, 동적 공간
  
  * C++에서는 데이터를 저장해 두기 위한 메모리를, 대입하는 방법에 따라 자동공간(automatic storage), 정적 공간(static storage), 동적 공간(dynamic strogae)으로 구분한다.
  
    * 동적 공간은 **자유 공간 또는 힙(heap)**이라고도 부른다.
  
    * 각각의 메모리 공간에 대입된 **데이터 객체는 수명이 서로 다르다.**
  
      
  
    * #### 자동 공간(automatic storage)
  
      * 자동 공간을 사용하는 함수 안에서 정의되는 보통의 변수들을 **자동 변수(automatic variable)라고 한다.**
      * 자동 변수들이 자신이 정의되어 있는 함수가 호출되는 순간에 자동으로 생겨나 그 **함수가 종료되는 시점까지만 존재한다는 것**을 뜻한다.
      * **자동 변수는 자신들을 포함하고 있는 블록 안에서만 유효하다.**
      * 블록이란 중괄호로 묶여진 코드의 일정 부분을 말한다.
        * 블록에 변수를 선언하면, 그 변수는 프로그램이 **블록 안에 있는 구문들을 실행되는 동안에만 존재**한다.
    * **자동 변수는 스택에 저장된다.**
        * 메모리상의 스택에 그 값이 순차적으로 저장되고, 역순으로 해제되는 것을 의미한다.(후입선출(last-in, first-out) or LIFO)
      * 스택은 프로그램이 실행되는 동안 늘었다 줄었다를 반복한다.
  
  * #### 정적 공간(static storage)
  
    * 프로그램이 실행되는 동안에 **지속적으로 존재하는 공간**이다.
  
      * 변수를 정적으로 만드는 방법은 두 가지이다.
  
      1. 함수의 **외부에서 변수를 정의**한다.
  
      2. 변수를 선언할 때 **static 키워드**를 붙인다.
  
         - ```cpp
             static double fee = 56.50;
             ```
  
      * K&R C에서는 정적 배열과 정적 구조체만을 초기화할 수 있었지만, ANSI C에서는 자동 배열과 자동 구조체를 초기화하는 것을 허용한다.
  
    * #### 동적 공간(dynamic storage)
  
      * new와 delete 연산자는 보다 융통성 있는 방법을 제공한다.
      * 이들은 **자유 공간(free store)**이라 부르는 **메모리 풀(memory pool)을 관리**한다.
          * 이 풀은 **자동 변수와 정적 변수가 사용하는 메모리와 분리**되어 있다.
      * new와 delete는 어떤 함수에서 **메모리를 대입**하고, **다른 함수에서 해제**할 수 있도록 해준다.
          * 데이터의 수명은 프로그램의 수명이나 함수의 수명에 얽매이지 않는다.
      * new와 delete를 함께 사용하면 일반 변수를 사용할 때보다 메모리에 대해 더 강력한 제어권을 가질 수 있다.
      * 스택의 경우, 자동적으로 늘어나고 줄어드는 매커니즘으로 인해 값들이 **메모리 상에서 인접하여 존재**하게 된다.
        * 그러나, new와 delete의 상호작용으로 여유 공간을 확보할 수 있고, 더 복잡한 메모리 대입 요청을 처리할 수 있다.
  
  
  
  * ### 스택, 힙, 메모리 누수
  
    * new 연산자로 자유공간(힙)에 변수를 생성한 후 delete를 사용하여 이를 해제하지 않는다면?
      * 자유 공간에 **동적으로 대입된 변수나 데이터 객체**는 지시하는 포인터를 포함하고 있는 메모리가 사용 범위와 객체 수명 규칙 때문에 해제된다고 하더라도, **delete가 호출되지 않는 한 계속 살아남게 된다.**
      * 이러한 경우에 자유공간에 생존하고 있는 변수나 데이터 객체에 접근할 수 있는 방도가 전혀 없다.
        * 왜냐하면 변수나 데이터 객체가 저장되어 있는 메모리 위치를 지시하는 포인터가 이미 사라졌기 때문이다.
    * 이를 **메모리 누수(memory leak)**라고 한다.
      * **누수된 메모리는 프로그램을 실행하는 동안 내내 사용할 수 없게 된다.**
      * 메모리가 대입은 되었으나 **접근이 불가능**하다.
      * 극단적인 경우에, 메모리 누수가 매우 심각하여 애플리케이션에 할애된 **메모리를 모두 소진할 수도 있다.**(메모리 부족으로 프로그램이 먹통이 된다.)
      * 메모리 누수는 동일한 메모리 공간에서 운영되는 **운영 체제나 다른 애플리케이션에도 부정적인 영향**을 끼친다.
      * 메모리 누수를 피하려면, **new와 delete 연산자를 가능한 가깝게 결합하여 사용하는 습관**을 들여야 한다.
        * 자유 공간에 동적으로 변수나 데이터 객체를 대입하자마자 delete 연산자를 사용하 계획을 서둘러야 한다.





* ## 변수형의 조합

  * ### 구조체

    * ```cpp
      struct antarctica_years_end
      {
          int year;
      };
      ```

    * ```cpp
      antarctica_years_end s01, s02, s03;	//s01, s02, s03은 모두 구조체이다.
      ```

    * 멤버십 연산자를 사용하는 **멤버에 접근**할 수 있다.

      * ```cpp
        s01.year = 1998;
        ```

    * 구조체에 대한 **포인터를 생성**할 수 있다.

      * ```cpp
        antarctica_years_end * pa = &s02;
        ```

    * 포인터가 유효한 주소를 가리킬 경우, **간접적인 멤버십 연산자를 사용하여 멤버에 접근**할 수 있다.

      * ```cpp
        pa->year = 1999;
        ```

    * **구조체들의 배열을 생성**할 수 있다.

      * ```cpp
        antarctica_years_end trio[3];	//3개 구조체들의 배열이다.
        ```

    * 멤버십 연산자를 사용하여 **원소의 멤버들에 접속**할 수 있다.

      * ```cpp
        trio[0].year = 2003;	//trio[0]는 구조체이다.
        ```

      * trio는 배열이나, trio[0]는 구조체이다.

        * trio[0].year는 그 구조체의 멤버이다.

        * 배열 이름이 포인터이므로, 간접 멤버십 연산자를 사용할 수 있다.

        * ```cpp
          (trio+1)->year = 2004;	//trio[1].year = 2004와 동일하다.
          ```

    * **포인터 배열을 생성**할 수 있다.

      * ```cpp
        const antarctica_years_end * arp[3] = {&s01, &s02, &s03};
        ```

      * arp가 포인터 배열이면 arp[1]은 반드시 포인터이기 때문에 포인터의 간접 멤버십 연산자를 사용하여 멤버에 접근할 수 있다.

        * ```cpp
          std::cout << arp[1]->year << std::endl;
          ```

    * **배열에 대한 포인터를 생성**할 수 있다.

      * ```cpp
        const antarctica_years_end ** ppa = arp;
        ```

      * arp는 **배열의 이름**이므로, **첫 번째 원소의 주소**이다.

      * 첫 번째 원소는 포인터이므로, ppa는 const antarctica_years_end에 대한 포인터의 포인터 즉, **가 되어야 한다.

      * 이 선언을 혼합할 수 있는 방법이 매우 다양하다.

        * const를 빼먹고 하나의 순서 뒤바뀜 함수인 *나 두 개의 **를 잊어버리거나 다른 구조형과 혼합할 수도 있다.

        * C++11 버전의 auto 명령어의 예가 있다.

          * 이 컴파일러는 완벽하게 어떤 타입의 arp인지 알고서, 사용자에게 정확한 변수 형을 알려줄 수 있다.

          * ```cpp
            auto ppb = arp;	//C++11에서 자동으로 변수형을 확인한다.
            ```

      * ppa는 구조체의 포인터를 가리키는 포인터이며, *ppa는 구조체를 가리키는 포인터이다.

        * 그래서 간접적인 멤버십 연산자를 이용하여 데이터에 접근할 수 있다.

        * ```cpp
          std::cout << (*ppa)->year << std::endl;
          std::cout << (*(ppb+1))->year << std::endl;
          ```

        * ppa는 arp의 첫 번째 멤버를 가리키는 포인터이기 때문에, *ppa는 그 첫 번째 멤버로 &s01이다.

          * 그러므로, (*ppa)->year는 s01의 year 멤버이다.

        * 두 번째 구문에서 ppb+1은 다음 요소인 arp[1]을 지목하는데, 그것이 &s02이다.

        * 괄호는 정확한 관계를 얻기 위해서 필요하다.

          * *ppa->year는 * 운영자를 ppa->year에 적용하려는 시도를 하는데, year 멤버가 포인터가 아니기 때문에 성공할 수 없다.

  

  * ### 배열의 대안

    * #### Vector 템플릿 클래스

      * vector 템플릿 클래스는 **동적 배열에 속하는 string 클래스와 유사**하다.

      * 프로그램이 실행되는 동안 **vector 객체의 크기를 세팅**할 수 있고, **새로운 데이터를 마지막에 추가하거나 중간에 데이터를 삽입**할 수도 있다.

      * 기본적으로 **동적 배열을 생성하기 위해 new를 사용하는 것을 대체**할 수 있다.

      * vector 클래스는 메모리를 관리하기 위해서 **new와 delete를 사용하지만, 그 과정은 자동으로 진행**된다.

      * ##### 문제점

        1. vector 객체를 사용하기 위해서는 **vector 헤더 파일을 포함**해야 한다.
        2. vector 식별자는 std 이름 공간의 일부분이기 때문에 using 명령, using 선언 또는 std::vector를 사용할 수 있다.
        3. 템플릿은 **저장된 데이터 형태를 지시하기 위해서 다른 구문을 사용**한다.
        4. vector 클래스는 **원소의 개수를 지칭하기 위해서 다른 구문을 사용**한다.

    * #### array 템플릿 클래스(C++11)

      * 사용자가 **고정된 크기의 배열만 필요**하다면, **내재 배열형을 사용하는 것이 효과적**이다.
        * 그러나, **안전성과 편리성은 다소 줄어든다.**
      * C++11은 그 **array 템플릿 클래스를 더해** 줌으로써 이러한 경우에 대해 **해결방안을 제시**한다.
        * 이것은 std 이름 공간의 일부분에 해당된다.
      * array 객체는 자유 저장 대신에 고정된 크기의 고정 메모리 대입을 사용하여 내재 배열이 지닌 것과 동일한 수준의 효율성을 지닌다.
      * array 객체를 생성하기 위해서는 array 헤더 파일을 포함시켜야 한다.

    * ### 배열, Vector 객체, Array 객체 비교

      * 내재 배열, vector 객체, 또는 array 객체를 사용하건, 사용자는 개별 멤버에 접속하기 위해서는 표준 배열 표식을 사용할 수 있다.

      * 주소를 통해서 **array 객체가 동일한 지역의 메모리를 사용**한다.(이 경우에는 stack)

        * vector 객체는 다른 지역에 저장된다.(자유 저장 또는 heap)

      * 하나의 array 객체를 또 다른 array 객체에 대입할 수 있다.

        * 내재 배열에 대하여 데이터를 요소별로 복사해야 한다.

        * ```cpp
          a1[-2] = 20.2;
          *(a1-2) = 20.2;	//위 구문과 같다.
          ```

          * a1 포인터는 두 개의 이중 요소를 뒤로 이동시킨다.
          * 20.2를 그곳에 위치시켜서 그 정보가 배열의 외부에 있는 한 지역에 그 정보를 저장한다.
          * C와 마찬가지로 **C++은 범위 밖 에러에 대하여 체크하지 않는다.**
          * 이것이 내재 배열의 불안전한 행위의 한 예이다.

        * at() 멤버 함수를 사용하여 방지할 수 있다.

          * ```cpp
            a2.at(1) = 2.3;	//2.3을 a2[1]에 대입함.
            ```

          * bracket notation을 사용하는 것과 at() 멤버 함수를 사용하는 것의 차이

            * 만약 사용자가 at()를 사용할 경우 유효하지 않은 인덱스가 **런타임과 프로그램 도중에  디폴트에 의해 잡히게 되어 전격 취소**된다.
            * 체크를 늘리려면 러닝 타임을 늘려야 가능하기 때문에, **C++에서는 두 가지 notation을 사용할 수 있는 옵션을 제공**하고 있다.
