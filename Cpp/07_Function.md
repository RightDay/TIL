# 함수 - C++의 프로그래밍 모듈

- 함수를 사용하려면 함수 정의와 함수 원형이 필요하고, 함수 호출을 사용해야 한다.

- 함수 정의

  - 무엇을 하는 것인지를 실제로 구현해 놓은 코드이다.

- 함수 원형

  - 함수에 어떠한 종류의 값이 몇 개 전달되어야 하며, 함수로부터 어떤 종류의 리턴값을 돌려받는지를 서술하는 함수의 인터페이스이다.

- 함수 호출

  - 프로그램에서 함수에 매개변수를 전달하여, 프로그램의 제어를 함수에 넘기는 것이다.

- C++는 매개변수를 함수에 값으로 전달한다.

  - 함수 정의에 있는 형식 매개변수는 함수 호출이 제공하는 값으로 초기화되는 새로운 변수이다.
  -  C++ 함수는 데이터의 복사본을 가지고 작업하게 되므로 원본 데이터는 보호된다.

- 배열 이름 매개변수를 첫 번째 배열 원소의 주소로 간주한다.

  - 기술적으로 볼 때, 포인터는 원본 주소의 복사본이기 때문에 여전히 값으로 전달된 것이다.

  - 그러나 그 함수는 그 포인터를 사용하여 배열의 원본에 접근한다.

  - 다음 두 선언은 함수의 형식 매개변수를 선언할 때 동등하다.

    - ```cpp
      typeName arr[]
      typeName * arr
      ```

      - arr가 typeName을 지시하는 포인터라는 것을 의미한다.
      - 함수의 코드를 작성할 때, arr를 배열 이름처럼 사용하여 원소에 접근할 수 있다.(arr[i]).
      - 포인터를 전달할 때 형식 매개변수를 const형을 지시하는 포인터로 선언하면, 원본 데이터를 보호할 수 있다.
      - 배열의 주소를 전달하는 것은 배열 크기에 대한 정보를 함께 전달하지 않는다.
      - 일반적으로 배열의 크기는 별도의 매개변수로 전달해야한다.
      - STL의 알고리즘에서와 같이 범위를 지정해 주기 위해서 배열의 시작과 끝 포인터를 보내는 방법도 있다.

- C++에서는 C 스타일의 문자열을 세 가지 방식으로 표현한다.

  - 문자배열, 문자열 상수, 문자열을 지시하는 포인터를 사용하여 표현한다.
    - 이들은 모두 문자를 지시하는 포인터, 즉 char *형이다.
  - C++는 문자열의 끝을 나타내기 위해 널 문자(\0)를 사용한다.
  - 문자열 함수는 처리하고 있는 문자열의 끝을 판단하기 위해 문자를 사용한다.

- C++는 구조체를 기본 데이터형처럼 취급한다.

  - 구조체도 함수의 값으로 전달할 수 있으며, 함수의 리턴형으로 사용할 수 있다.
  - 구조체의 덩치가 클 경우에는 구조체를 지시하는 포인터를 함수에 전달하는 것이, 원본 데이터를 가지고 작업할 수 있으므로 더 효율적이다.
  - 동일한 내용이 classic 객체에도 적용된다.

- C++ 함수는 재귀적으로 자기 자신을 호출할 수 있다.

  - 함수의 특정 부분에 자기 자신을 호출하는 코드가 포함될 수 있다.

- C++ 함수의 이름은 함수의 주소처럼 행동한다.

  - 함수를 지시하는 포인터를 매개변수로 사용하여, 호출하기 원하는 두 번째 함수의 이름을 첫 번째 함수에 전달할 수 있다.
